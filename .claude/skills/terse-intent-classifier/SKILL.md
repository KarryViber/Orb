# 短指令作用层判定协议

> 来源：04/17、04/18、04/19 连续 3 天夜间反思升级行动项。根因：短输入省略了"作用对象"和"作用层"，直接按最新话题执行容易错层（把记忆层的事当运行层处理，或把架构层的事当配置层处理）。

## 触发条件

消息 ≤15 字 且 满足其一：

- **指代词**：含「这个 / 那个 / 上面 / 刚才 / 它 / 那条 / 这条」
- **动词型短指令**：以「改 / 删 / 切 / 跑 / 停 / 加 / 去掉 / 补 / 关 / 开」开头
- **元问句**：含「更优雅 / 更好 / 有没有办法 / 有没有更 / 能不能」

同时满足「thread 历史里有多个候选对象」时强制触发；thread 内只有唯一明显对象可以跳过。

## 动作

**必须先输出一行作用层判定，再动手。**

格式严格：

```
[层] {记忆层 | 配置层 | 运行参数层 | 外部动作层 | 架构层}
[对象] {具体文件 / 具体规则 / 具体行为}
[下一步] {执行 | 先澄清 | 写进记忆}
```

## 层的定义

| 层 | 典型对象 | 典型短指令 |
|---|---|---|
| **记忆层** | `~/.claude/projects/.../memory/*.md`、`~/Orb/profiles/karry/data/lessons/*.md`、holographic facts | 「记住这个」「忘掉那条」「改一下那条规则」 |
| **配置层** | `~/Orb/config.json`、cron-jobs.json、`.claude/settings.json`、hooks | 「把这个 cron 关掉」「加个权限」「改一下 trigger」 |
| **运行参数层** | 当前任务的 model/effort、--append-system-prompt、单次会话行为 | 「跑一下」「切到 sonnet」「再试一次」 |
| **外部动作层** | Slack 发送、API 调用、文件对外交付、git push | 「发出去」「推上去」「确认」 |
| **架构层** | `~/Orb/src/*`、adapter 接口、IPC 协议、lib/* | 「改一下这个架构」「换种方式实现」「重构这块」 |

## 判断不明 → 直接反问

不猜。反问模板（≤30 字）：

> 你指的是 A（{层}/{对象}）还是 B（{层}/{对象}）？

## 反例

**错（04/18）**：
- 用户：「有哪些值得动手」（在 cron 输出 thread 下）
- 错误理解：按工作项目维度作答
- 正确理解：[层] 运行参数层；[对象] 本 cron 产出的行动项；[下一步] 执行

**错（首见 04/17）**：
- 用户：「改一下」
- 错误做法：直接按最新消息推断改什么
- 正确做法：[层] ?；[对象] ?；[下一步] 先澄清
